Security and release policy

No public app source. No public release tags. No source archives.

GitHub automatically generates source archives for tagged releases, so Luvi avoids public release tags for installer distribution.

Public repo contents

The repository contains only this website, README, and static assets needed by the website.

Installer policy

Installer binaries are not committed to git and are not published through tagged GitHub Releases.

Update policy

The desktop app uses signed update metadata from the configured private channel.

Access policy

Team requests only target existing Luvi users, and accepted users receive access inside the app.
