Workflow guide

From clean workspace to running team system.

Luvi’s workflow is intentionally simple: create, invite, coordinate, review, transfer when ownership changes.

01

Create a project

The team leader creates a Luvi project. A dedicated project chat group is created with a unique id.

02

Add existing users

Only users already registered in Luvi can receive team requests. They accept inside the app inbox.

03

Run the board

Use Kanban for movement, list for scanning, and table for direct editing of priorities and due work.

04

Coordinate in chat

Project rooms hold shared context. Direct messages handle person-to-person sync without crowding the project room.

05

Transfer ownership

When a project moves to another Luvi user, project data and chat ownership move with it.
